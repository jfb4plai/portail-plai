⏧ 5 minutes à lire 🎯 Le pourquoi avant le comment
Le problème central

Claude Code est capable d'écrire du code complet, de refactoriser des modules, de générer des tests. Ce n'est pas la capacité qui pose problème — c'est la discipline. Livré à lui-même, Claude Code saute la phase de conception, plonge directement dans le code, et rationalise chaque raccourci avec confiance.

Pour des applis ponctuelles ou des scripts jetables, c'est acceptable. Pour des outils livrés à des enseignants qui ne signaleront pas un bug subtil — ils abandonnent l'outil — c'est insuffisant.

Sans méthode
Claude fonce, produit quelque chose qui marche en démo. Les cas limites que les enseignants vont rencontrer ne sont pas testés. Tu le découvres deux semaines après la livraison.
Avec Superpowers
Claude ne peut pas coder avant que les comportements attendus soient définis et approuvés. Ce que tu as validé dans le plan est ce que l'appli fait.
Ce qu'est Superpowers — et ce qu'il n'est pas

Superpowers est un plugin open source (licence MIT) créé par Jesse Vincent. Il fonctionne comme un ensemble de fichiers markdown — appelés skills — que Claude Code lit et applique comme instructions de workflow. Ce n'est pas un prompt amélioré. C'est une méthodologie encodée en code.

✅
C'est
Une méthodologie complète : brainstorming → plan → TDD → revue → livraison. Chaque phase a des portes qui ne s'ouvrent que si la précédente est validée.
❌
Ce n'est pas
Un correcteur de mauvaises spécifications. Si tu décris mal ton appli au départ, Superpowers construit rigoureusement la mauvaise chose.
⚙️
Prérequis
Claude Code CLI (abonnement Pro ou Max). Superpowers s'installe en une commande dans le terminal. Aucun hardware spécifique requis.
⚠️ Honnêteté sur le coût

Superpowers ajoute ~480 tokens par échange et ~29 500 tokens quand un skill se déclenche. La phase brainstorming + plan prend 10 à 20 minutes de plus qu'un démarrage immédiat. Cet investissement est rentabilisé sur les applis complexes — pas sur les scripts de 30 lignes.

🏫 Pour les enseignants
Ce que Superpowers change pour vos écoles
Vous ne coderez probablement pas vous-mêmes — mais vous serez les utilisateurs finals. Voici ce que ce plugin garantit concrètement.
⏧ 3 minutes 🎯 L'essentiel sans jargon
L'analogie qui résume tout

« C'est comme si vous demandiez à un élève de rendre un exercice. Sans méthode : il écrit, soumet, espère. Avec Superpowers : il doit d'abord écrire les critères de réussite (le test), puis rédiger sa réponse, puis vérifier que ça colle. L'IA ne peut pas tricher l'ordre. »

Pourquoi c'est important pour vous

Si un collègue, un étudiant en stage, ou l'équipe numérique de votre école utilise Claude Code pour créer des outils pédagogiques (quiz automatiques, générateurs de fiches, extracteurs de données), Superpowers garantit que le résultat est fiable et maintenable — pas seulement impressionnant à la démo.

Sans Superpowers
Code qui fonctionne une fois sur deux. Corrections qui cassent autre chose ailleurs. Solutions impossibles à comprendre ou modifier plus tard.
Avec Superpowers
Workflow prévisible. Outils testés avant livraison. Comportements que les enseignants vont déclencher (champs vides, mauvais format, double-clic) vérifiés à l'avance.
Le cycle TDD expliqué sans jargon

Au cœur de Superpowers se trouve une méthode appelée TDD (Test-Driven Development). Elle impose un ordre précis que l'IA ne peut pas contourner :

🔴
Rouge — définir ce que l'outil doit faire
Écrire un test qui décrit le comportement attendu — avant d'écrire le code. Ce test échoue (normal : le code n'existe pas encore). Exemple : on définit ce que doit contenir une grille d'évaluation en boucherie avant de la coder.
🟢
Vert — écrire le minimum de code pour valider
Écrire uniquement ce qui est nécessaire pour que le test réussisse. Pas d'improvisation, pas d'extras non spécifiés.
🔁
Refactor — nettoyer sans casser
Améliorer le code sans modifier son comportement. Les tests vérifient que rien n'a changé pour l'utilisateur.
Ce que Superpowers n'est pas
🖱️
Pas un outil direct pour vous
Pas d'interface graphique, pas de bouton à cliquer. C'est un outil pour la personne technique qui développe les applis de votre école.
🎁
Pas gratuit
Nécessite un abonnement Claude Code (Anthropic). À prévoir dans le budget numérique de l'école si des outils sont développés en interne.
🧠
Pas magique
Structure le travail du développeur IA. Ne remplace pas le jugement humain — et surtout pas votre expertise pédagogique sur ce que doit faire l'outil.
✅ Message à retenir

Si votre école utilise Claude Code pour développer des outils numériques pédagogiques, insistez pour que l'équipe technique installe Superpowers. Cela garantit que les outils livrés sont fiables, testés et compréhensibles — pas juste impressionnants à la démo.

💡
Pourquoi la plupart des commandes sont "auto"
Le design fondamental de Superpowers
▼

Superpowers se charge via un hook SessionStart qui s'exécute à chaque ouverture de session Claude Code. Il injecte une règle fondamentale dans le contexte : si même 1% de chance qu'un skill s'applique à la tâche en cours, Claude doit vérifier et l'invoquer.

Cela signifie que tu n'as pas à mémoriser les commandes pour les déclencher — Claude le fait automatiquement quand il détecte le contexte approprié. Tu commandes le quoi (décrire l'appli), Superpowers gère le comment (quelle méthode appliquer).

En pratique

Sur une session typique de création d'appli, tu n'utiliseras activement que /brainstorming au départ et /requesting-code-review avant de livrer. Les 12 autres commandes se déclenchent seules.

🚀 Démarrer
Flux minimal pour une nouvelle appli
Les 5 étapes concrètes pour démarrer un projet avec Superpowers — sans avoir à tout mémoriser.
⏧ 20 min de plus que sans Superpowers 🎯 Récupéré sur la correction de bugs
Les 5 étapes dans l'ordre
1
Ouvre Claude Code dans le dossier de ton projet
Dans PowerShell : cd C:\chemin\vers\ton\projet puis claude. Superpowers se charge automatiquement au démarrage de la session.
2
Lance le brainstorming — décris l'appli
Tape /brainstorming puis décris ce que tu veux créer. Claude pose des questions progressives sur le contexte, les utilisateurs (les enseignants), les cas limites. Ne code rien encore.
3
Valide le document de spécification
Superpowers génère un document de design. Lis-le attentivement — c'est ici que tu évites de construire la mauvaise chose. Approuve ou corrige. Claude ne peut pas passer à l'étape suivante sans ton accord explicite.
4
Valide le plan d'implémentation
Superpowers génère automatiquement un plan avec les tâches de 2 à 5 minutes, les fichiers à toucher, les tests à écrire. Relis chaque tâche — le plan a des cases à cocher qui servent de journal de progression si la session plante.
5
Laisse Superpowers exécuter — interviens si nécessaire
Les sous-agents implémentent le plan tâche par tâche, avec TDD et revue automatique à chaque étape. Tu n'interviens que si une décision architecturale nécessite ton jugement. Avant de livrer : /requesting-code-review.
Comparaison avec et sans Superpowers sur une session type
Sans
Tu décris l'appli → Claude code immédiatement → tu testes → tu signales les bugs → Claude corrige et en crée d'autres → tu retestes → cycle sans fin jusqu'à livraison approximative.
Avec
Tu décris l'appli → brainstorming (10-15 min) → plan validé → exécution structurée → revue automatique → livraison correspondant à ce qui a été spécifié.
Checklist avant de livrer aux enseignants
Brainstorming validé — le document de spécification a été relu et approuvé avant le début du code
Tous les tests passent — le cycle rouge-vert-refactor a été respecté pour chaque fonctionnalité
Revue de code faite — /requesting-code-review a produit un résultat sans problème bloquant
Cas limites testés — les comportements que les enseignants non-techniques déclencheront ont été vérifiés (champs vides, mauvais format, double-clic…)
Session de test réelle — l'appli a été testée par quelqu'un qui ne sait pas comment elle fonctionne, pas uniquement par son créateur
Erreurs classiques à éviter avec Superpowers
❌
Approuver le plan sans le lire
La faute la plus fréquente
▼

Superpowers génère un plan détaillé — mais c'est toi qui valides. Si tu approuves sans lire, tu perds l'avantage principal du système. La valeur est dans la phase planning : c'est là que les problèmes architecturaux sont détectés avant d'être codés.

Règle pratique

Lis chaque tâche du plan et vérifie qu'elle a bien un test associé avant une implémentation. Si une tâche n'a pas de test, le plan est incomplet — demande à Claude de le corriger.

❌
Utiliser Superpowers pour des scripts simples
Outil disproportionné
▼

Pour un script de 30 lignes, un one-shot, ou une exploration rapide d'une idée, Superpowers ajoute une friction inutile. Le plugin est conçu pour des applis avec plusieurs fonctionnalités, des utilisateurs non-techniques, et un besoin de maintenabilité.

Disproportionné
Script de conversion CSV, test rapide d'une API, exploration d'une bibliothèque. Pas besoin de Superpowers.
Approprié
Quiz interactif pour 200 élèves, générateur de fiches pédagogiques, outil de suivi des compétences. Superpowers a du sens.
❌
Croire que Superpowers remplace la spécification
Le piège de la délégation totale
▼

Superpowers structure l'implémentation — il ne remplace pas ton jugement sur ce que doit faire l'appli. Si tu entres en brainstorming avec une idée vague ("faire un truc pour les quiz"), Superpowers va poser des questions — mais les réponses, c'est toi qui les donnes. Une mauvaise spécification reste une mauvaise appli, même bien construite.

⚠️ À mémoriser

Avant de lancer /brainstorming, passe 5 minutes à répondre à ces questions toi-même : qui sont les utilisateurs exacts ? Quel est le cas d'usage principal ? Qu'est-ce qui doit absolument fonctionner, même si le reste est imparfait ?

« Superpowers ne rend pas Claude plus intelligent — il le rend constant. Et dans le développement d'applis pour des enseignants, la constance bat l'intelligence à chaque fois. »

⚙️ Installer
Installer Superpowers sur Windows
Guide complet testé et validé — de zéro à Superpowers actif en 10 minutes.
⏧ 10 minutes 🪟 Windows 11 · PowerShell 7 · Claude Code v2.1.156
Prérequis
✅
Abonnement Claude Pro ou Max
Requis pour accéder à Claude Code CLI. À activer sur claude.ai avant de commencer.
🌐
Chrome installé (obligatoire)
Edge bloque l'authentification OAuth d'Anthropic. Chrome doit être le navigateur utilisé pour l'autorisation.
💻
PowerShell 7
Pas PowerShell 5, pas le terminal Windows. Chercher "PowerShell 7" dans le menu Démarrer.
🔐
Claude ouvert dans Chrome
Se connecter sur claude.ai dans Chrome AVANT de lancer le terminal — sans cette étape, l'OAuth échoue.
⚠️ Action préalable obligatoire

Ouvre Chrome et connecte-toi sur claude.ai avec ton compte avant d'ouvrir PowerShell. Sans cette étape, l'authentification OAuth échoue avec le message "Invalid code".

Installation pas à pas
1
Ouvrir PowerShell 7
Appuie sur Windows + S, tape PowerShell 7 et ouvre-le. Tu dois voir PS C:\Users\…> comme invite de commande.
2
Ajouter Claude Code au PATH (si nécessaire)
Si la commande claude n'est pas reconnue : $env:PATH += ";C:\Users\TON_NOM\.local\bin". Si Claude Code n'est pas encore installé : irm https://claude.ai/install.ps1 | iex
3
Lancer Claude Code et s'authentifier
Tape claude. Sélectionne "1. Claude account with subscription". Une URL OAuth s'affiche — Ctrl+clic pour l'ouvrir dans Chrome. Clique sur "Authorize". Succès : "Welcome back !" avec ton email et Claude Pro.
4
Installer Superpowers
Dans la session Claude Code, tape : /plugin install superpowers@claude-plugins-official. Sélectionne "Install for you (user scope)" — active Superpowers pour TOUS tes projets futurs.
5
Activer Superpowers
Tape /reload-plugins. Succès : "Reloaded: 1 plugin · 6 agents · 1 hook". À partir de maintenant, Superpowers se charge automatiquement à chaque nouvelle session.
Problèmes courants
⚠️
"claude" n'est pas reconnu
PATH manquant
▼

Ajoute le chemin manuellement dans PowerShell :

$env:PATH += ";C:\Users\TON_NOM\.local\bin"
Pour rendre permanent

[Environment]::SetEnvironmentVariable("PATH", $env:PATH + ";C:\Users\TON_NOM\.local\bin", "User")

⚠️
"OAuth error: Invalid code"
L'URL s'est ouverte dans Edge
▼

L'URL OAuth s'est ouverte dans Edge au lieu de Chrome. Copie l'URL affichée dans le terminal et colle-la manuellement dans Chrome.

⚠️
"/plugin isn't available"
Mauvaise interface
▼

Tu es dans l'application bureau graphique Claude, pas dans le terminal CLI. Les commandes /plugin ne fonctionnent que dans PowerShell avec la commande claude.
